# Security Baseline Checklist v0.8

Project: First Nations Governance Commons  
Target: public demo, Decidim sandbox, future controlled pilot

## Public Static Demo

- [ ] Fake/sample data only.
- [ ] No secrets in files.
- [ ] No real member data.
- [ ] No cultural material.
- [ ] No analytics tracking without review.
- [ ] No forms collecting real personal data.
- [ ] Public demo warning visible.

## Decidim Fake-Data Sandbox

- [ ] HTTPS enabled if public.
- [ ] Admin accounts named and controlled.
- [ ] No shared admin credentials.
- [ ] Strong passwords.
- [ ] MFA enabled if available.
- [ ] Demo users only.
- [ ] No real member/community/cultural data.
- [ ] SMTP configured safely.
- [ ] Backups configured if hosted.
- [ ] Restore process known.
- [ ] Uploads checked for fake/sample status.
- [ ] Provider/developer documents hosting and access.

## Future Real Pilot

Do not proceed unless:

- [ ] Authorising body confirmed.
- [ ] Governance Charter approved.
- [ ] Data Sovereignty Charter approved.
- [ ] Privacy notice reviewed.
- [ ] Cultural safety rules approved.
- [ ] Admin access rules approved.
- [ ] Hosting jurisdiction approved.
- [ ] Retention/deletion rules approved.
- [ ] Breach response process approved.
- [ ] Offline participation process approved.
- [ ] Local admins trained.

## Incident Response Basics

Before any real data:

- [ ] Name incident contact.
- [ ] Define breach triage process.
- [ ] Define who must be notified.
- [ ] Define how evidence is preserved.
- [ ] Define how public statements are approved.
- [ ] Define how affected participants are supported.

## High-Risk Materials

Do not upload unless explicit approval exists:

- Cultural material.
- Sacred/restricted knowledge.
- Gender-restricted knowledge.
- Member lists.
- Identity documents.
- Health/legal/casework records.
- Child safety information.
- Precise sensitive location data.
