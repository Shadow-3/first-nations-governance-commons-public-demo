# Toastmasters First Nations Private Pilot Implementation Plan v2.8

Mode: controlled sample/test preparation only  
Sample initiative: Toastmasters First Nations  
Initial developer/custodian: Barayamal / Dean Foley  

## What This Plan Does

This plan prepares a controlled First Nations Toastmasters private pilot without turning the public demo into a real data collection system.

The public demo can explain the model, show fake-data workflows, support partner review, and generate implementation evidence. Real records belong only in the approved private environment.

## Current Public Facts To Respect

The Barayamal Toastmasters First Nations page describes a proposed online club in formation, a 20 founding member charter target, initial sponsorship, a demo meeting, and the boundary that official membership starts only after official Toastmasters paperwork, payment or sponsorship reconciliation, and charter acceptance.

Official Toastmasters sources also separate the start-a-club pathway, official application, charter paperwork, member recruitment, member dues, and governing documents from this public Governance Commons demo.

## Pilot Tracks

### Founding Member Now

Use when a person is ready to be considered for the charter group after official applications, sponsorship, and Toastmasters acceptance.

Private records may include contact details, readiness track, age confirmation, officer interest, membership type, consent timestamp, and approved status fields.

### Demo First

Use when a person wants to attend the online demo before deciding.

Private records should stay lighter: contact, availability, access support request, consent, and follow-up preference.

### Sponsor, Mentor, Or Supporter

Use when an existing Toastmasters member, partner, or community supporter can help without being counted as a charter-ready founder.

Private records should focus on support role, contact, availability, and referral notes.

### Not Sure Yet

Use when a person wants information without being counted toward the charter pathway.

Private records should be minimal and should not trigger official paperwork.

## Real Intake Start Gate

Real intake should not start until:

- Pilot authority is confirmed.
- Private environment is approved.
- Minimum data schema is locked.
- Access register is approved.
- Consent and withdrawal wording is approved.
- Official Toastmasters handoff owner is named.
- Meeting link sharing route is private.
- Incident and pause process is rehearsed.
- Fake-data dry run is complete.
- Public evidence redaction checklist is ready.

## Official Toastmasters Boundary

Governance Commons can track:

- Redacted status summaries.
- Founder count categories.
- Demo-first count categories.
- Supporter count categories.
- Whether official handoff is pending, ready, submitted, or accepted.

Governance Commons must not publicly store:

- Official applications.
- Signatures.
- Member numbers.
- Payment or card records.
- Charter member roster with real names.
- Official Toastmasters account credentials.
- Official acceptance records containing private identifiers.

## Private Pilot Operating Rhythm

### Weekly

- Review new intake.
- Check founder count and track quality.
- Check support and access needs.
- Review access register.
- Confirm no public evidence breach.
- Update redacted public status only if safe.

### Before Demo Meeting

- Confirm private meeting link distribution.
- Confirm moderator and conduct wording.
- Confirm access support path.
- Confirm no public list of attendees.
- Confirm incident pause owner.

### After Demo Meeting

- Move demo-first contacts only if they actively choose the founding-member track.
- Do not infer consent from attendance.
- Record follow-up preference privately.
- Keep public reporting aggregate.

### Before Official Submission

- Confirm 20 prospective charter members.
- Confirm the non-dual member requirement status.
- Confirm minimum officer slate.
- Confirm applications and signatures through official Toastmasters process.
- Confirm sponsorship/payment reconciliation outside public demo.
- Confirm all public evidence is redacted.

## Done Definition

The private pilot preparation phase is done when:

- Required implementation actions are complete or approved.
- Required environment controls are complete or approved.
- Never-public data fields are separated.
- Access register is reviewed.
- Official handoff boundary is accepted.
- Dry run defects are closed or held with a named decision.
- Go/no-go decision is recorded.
- Public demo remains fake/sample only.
