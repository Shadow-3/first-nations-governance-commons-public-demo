# Authority Motion Register Template v3.5

## Purpose

Use this template in the private authority meeting to decide what is ready, held, blocked, or carried as conditions.

## Motions

- MOT-001: Accept public package as fake/sample safe. Owner: Meeting chair. Status: Ready. Evidence: Public redaction QA proof. Output: Public status can continue.
- MOT-002: Table Dean advisory note as advisory only. Owner: Meeting chair. Status: Ready. Evidence: Dean boundary note. Output: Advisory findings accepted as input.
- MOT-003: Accept Dean conflict/custody declaration. Owner: Pilot authority. Status: Required. Evidence: Conflict/custody note. Output: Advisory role accepted or varied.
- MOT-004: Approve authority quorum and delegation. Owner: Pilot authority. Status: Blocked. Evidence: Private authority record. Output: Go/hold/no-go authority exists.
- MOT-005: Approve IDS/data governance controls. Owner: Data reviewer. Status: Blocked. Evidence: Private IDS/data governance record. Output: Data classes and obligations approved.
- MOT-006: Approve privacy controls. Owner: Privacy reviewer. Status: Blocked. Evidence: Private privacy decision record. Output: Notice, consent, withdrawal, correction, incident route approved.
- MOT-007: Approve private environment and access. Owner: Technical custodian. Status: Blocked. Evidence: Environment proof and revoke test. Output: Private workspace can be activated.
- MOT-008: Confirm official Toastmasters boundary. Owner: Toastmasters launch lead. Status: Hold. Evidence: Official boundary memo. Output: Official records remain external or approved pathway exists.
- MOT-009: Accept conditions, expiry, monitoring, and closeout. Owner: Pilot authority. Status: Required. Evidence: Decision record shell. Output: Conditions can be carried into private activation.
- MOT-010: Record limited private pilot go/hold/no-go. Owner: Pilot authority. Status: Blocked. Evidence: All blocked motions closed or carried as approved conditions. Output: Final decision recorded privately.

## Decision record fields

- Motion ID.
- Decision: approve, hold, block, carry condition, or no-go.
- Decision owner.
- Evidence reference in private vault.
- Conditions.
- Expiry date.
- Stop trigger.
- Monitoring owner.
- Public summary wording.
