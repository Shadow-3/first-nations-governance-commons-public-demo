# P0 Outcome and Repeat-Run Protocol v5.3

## Outcome meanings

- Pass: accepted for the controlled fake/sample run only.
- Hold: owner review or evidence is incomplete.
- Repeat required: the test must be rerun before any next gate.
- No-go: the path is not appropriate unless authority changes.

## Blocking rule

Any P0 Hold, Repeat required, No-go, Missing evidence, Rejected evidence, or Open P0 issue blocks limited private pilot, provider real-record access, outbound email, official Toastmasters workflows, and live Decidim access.

## Proceed rule

Proceed to any limited private pilot remains blocked until every P0 outcome is Pass, every P0 evidence item is Accepted, every P0 issue is Closed, and required owners sign off privately.

## Repeat run trigger

Repeat the P0 stress lab whenever any P0 outcome is Hold, Repeat required, or No-go; any P0 evidence item is Missing, Rejected, or on Hold; or any P0 issue remains Open.
