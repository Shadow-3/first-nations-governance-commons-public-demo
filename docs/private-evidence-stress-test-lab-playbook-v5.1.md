# Private Evidence Stress Test Lab Playbook v5.1

Use this after v5.0 execution control and before any live pilot, real intake, outbound email, provider real-record access, or live Decidim access.

## Operating Rules

- Use v5.1 before any live pilot or provider handoff to stress-test the v5.0 execution controls under failure, rollback, privacy, support, official-boundary, and public-communications pressure.
- A control passes only when it survives a fake/sample failure drill, has accepted private evidence, names a rollback owner, and leaves no real participant, official Toastmasters, cultural, business-sensitive, credential, or private evidence data in the public package.
- No limited private pilot, real intake, provider real-record access, outbound email, or live Decidim access can proceed unless every P0 stress test and owner threshold passes privately.
- Public pages may show only fake/sample stress status, aggregate counts, approved hold/no-go wording, and learning summaries; they must not publish private reasons, minutes, evidence files, participant records, official records, support cases, credentials, or live access details.

## Run Order

1. Select each P0 stress scenario and rehearse the failure with fake/sample data only.
2. Require the named owner to accept or reject the private evidence.
3. Run the matching red-team, provider, rollback, and public wording drill.
4. Update the decision threshold register.
5. Hold, repeat, no-go, archive, or schedule the next private owner review.
