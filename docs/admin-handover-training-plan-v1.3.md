# Admin Handover And Training Plan v1.3

Date: 2026-06-04  
Initial developer/custodian: Barayamal / Dean Foley  
Mode: fake-data Decidim sandbox handover

## Purpose

This plan describes the minimum handover needed after the fake-data Decidim sandbox is built.

## Handover Audience

- Barayamal / Dean Foley.
- Any delegated platform admin.
- Any reviewer or support person who needs read-only access.

## Training Topics

### Admin Basics

- Login and admin dashboard.
- Organisation settings.
- Public pages.
- User roles.
- Moderation settings.

### Participation Setup

- Assemblies.
- Participatory processes.
- Proposals.
- Meetings.
- Surveys.
- Accountability/tracking.

### Safety And Data

- Fake-data-only boundary.
- What must never be uploaded.
- Public warning labels.
- How to remove sample content.
- How to report a mistaken upload.

### Operations

- Backup and restore path.
- Update responsibilities.
- Hosting/support contact.
- Known limitations.
- Issue escalation.

## Handover Checklist

- Admin credentials transferred safely.
- Roles documented.
- Backup approach documented.
- Support contact documented.
- Known risks documented.
- Acceptance test results shared.
- Fake-data labels confirmed.
- No real community records present.

## Training Format

Recommended format:

- 60-minute live walkthrough.
- 30-minute admin practice session.
- 30-minute Q&A.
- Written handover notes.

## Completion Rule

Handover is not complete until Barayamal or the delegated custodian can explain how to keep the sandbox fake-data only, update content, manage access, and request support.
