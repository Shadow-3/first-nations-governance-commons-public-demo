# Controlled Sandbox Test Lab Playbook v3.8

Use this playbook after the v3.7 Two-Track Pilot Onboarding model.

## Review finding

v3.7 separated Barayamal as the First Nations organisation onboarding test and First Nations Toastmasters as the controlled member tester. The remaining gap was operational: the project still needed fake accounts, scripted rehearsals, observer prompts, issue triage, decision gates, and evidence handling before any private pilot with real people.

## Operating rule

Use the Controlled Sandbox Test Lab only after v3.7 two-track onboarding is understood. Barayamal and First Nations Toastmasters must run as separate fake-data test tracks with separate fake accounts, observer notes, issue queues, evidence folders, go/hold/no-go gates, and closeout decisions.

## Lab sequence

1. Confirm the public fake-data rule.
2. Create or import only the sample accounts listed in the account matrix.
3. Run Barayamal scripts separately from First Nations Toastmasters scripts.
4. Record observer notes with public-safe prompts only.
5. Triage every critical and high issue before any real invite.
6. Use go/hold/no-go gates to close the rehearsal.
7. Keep private approval evidence outside this public package.
