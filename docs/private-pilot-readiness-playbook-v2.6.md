# Private Pilot Readiness Playbook v2.6

Release: v2.6 Private Pilot Readiness  
Mode: public template with fake/sample data only  
Initial developer/custodian: Barayamal / Dean Foley

## Review Finding

The v2.5 Organisation Onboarding package safely tests public coordination for First Nations Toastmasters, but it stops before the highest-risk transition point: moving from public sample mode into a controlled private pilot.

Without a private-pilot readiness layer, reviewers could understand that real data must stay private but still lack the operating controls for deciding when private records, official Toastmasters workflow, role access, consent, incident response, and closeout are ready.

## Upgrade

v2.6 adds Private Pilot Readiness: a control layer for deciding whether a First Nations organisation or initiative can move from public coordination into a private, approval-based test.

It does not collect real data. It defines the gates and evidence needed before any real private pilot starts.

## What v2.6 Adds

- Fourteen private-pilot readiness gates.
- Twelve test scenarios for intake routing, access support, official handoff, withdrawal, incident response, evidence, and closeout.
- Ten role/access rules.
- Twelve data-transfer rules.
- A browser-generated private pilot readiness brief.
- Document viewer coverage for the v2.6 docs, CSVs, and JSON package.

## Recommended Use

Use the v2.6 page after Organisation Onboarding and before any real private workflow.

The correct sequence is:

1. Keep public demo and public repository fake/sample only.
2. Use Organisation Onboarding to test public join/support language.
3. Use Private Pilot Readiness to test the private-pilot controls.
4. Approve privacy notice, consent, role access, data map, hosting, incident response, and closeout before any private records are collected.
5. Keep official Toastmasters paperwork, signatures, member numbers, payments, and charter submission in the official Toastmasters pathway.

## No-Go Triggers

Stop the pilot if:

- The public demo is treated as a real intake form.
- Real contact, access-need, payment, signature, member-number, official application, or private organisation contact data appears in the public demo, public GitHub repository, public screenshots, or fake-data sandbox.
- Organisation supporters are given authority over member data or official acceptance.
- Privacy notice, consent, role access, hosting, incident response, or closeout rules are missing.
- The private pilot is used as proof of community authority without separate First Nations governance approval.

## Required Evidence

Before a private pilot starts, retain evidence for:

- Named decision owner.
- Approved privacy notice.
- Approved minimum field list.
- Role/access matrix.
- Official Toastmasters pathway separation.
- Private environment setup.
- Incident response route.
- Withdrawal and deletion path.
- Redacted/fake QA evidence.
- Closeout decision rule.
