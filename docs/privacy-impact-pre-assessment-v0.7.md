# Privacy Impact Pre-Assessment v0.7

Project: First Nations Governance Commons  
Status: early planning tool, not legal advice  
Mode: public demo and Decidim sandbox planning

## Purpose

This pre-assessment identifies privacy and data risks before any real community pilot.

The public demo uses fake/sample data only. A real pilot would require a proper privacy and data review.

## Data Inventory

| Data Type | Public Demo | Decidim Sandbox | Real Pilot |
|---|---|---|---|
| Public pages | Fake/sample | Fake/sample | Real approved content |
| Account data | None | Demo users only | Real users possible |
| Member/eligibility data | None | None or fake | Sensitive, restricted |
| Proposals/comments | Fake/sample | Fake/sample | Real participation data |
| Votes/supports | Fake/sample | Fake/sample | Real decision data |
| Offline records | Fake/sample | Fake/sample | Sensitive depending content |
| Cultural material | None | None | Avoid or restrict |
| Partner responses | Fake/sample | Fake/sample | Real public or controlled records |
| Admin logs | Local/browser only | Real system logs | Real restricted logs |

## Questions Before Real Pilot

- What personal data is collected?
- What sensitive data is collected?
- What data is avoided entirely?
- Who can access each data category?
- Where is data hosted?
- Are backups encrypted?
- How long is data retained?
- Can participants request correction or deletion?
- What is public, member-only, restricted, redacted, or confidential?
- Is anonymous or pseudonymous participation allowed?
- Are children/youth participating?
- What happens if data is breached?

## Privacy Notice Must Explain

- Who runs the process.
- Why data is collected.
- What data is collected.
- How data is used.
- What may be published.
- Who can access data.
- How long data is kept.
- How to request correction/deletion.
- How to complain or ask questions.

## AI And Automation Guardrail

Do not use real community data, member data, cultural material, or restricted records to train AI systems unless explicitly approved by the relevant community authority.

AI-assisted summaries should be:

- Disclosed where appropriate.
- Human-reviewed.
- Kept out of restricted cultural material unless approved.
- Traceable back to original records.

## Pre-Assessment Outcome

| Question | Answer |
|---|---|
| Is public demo safe to share? | Yes, if it remains fake/sample data only. |
| Is sandbox safe to build? | Yes, if it remains fake/sample data only. |
| Is real pilot ready? | No. Needs privacy/data/cultural/governance approval. |

## Required Before Real Pilot

- Privacy notice.
- Data inventory.
- Hosting decision.
- Admin access rules.
- Retention/deletion schedule.
- Breach response process.
- Cultural safety approval.
- Data Sovereignty Charter sign-off.
