# Maintainer Charter v1.4

Initial maintainer: Barayamal / Dean Foley  
Mode: public demo and fake-data sandbox support

## Maintainer Duties

- Keep the public repository fake-data only.
- Review pull requests against the fake-data and community-authority boundary.
- Maintain public demo deployment quality.
- Keep package manifest, release notes, README, and package index current.
- Remove unsafe or misleading content quickly.
- Keep issue labels and triage lanes useful.
- Prepare the fake-data Decidim sandbox build handoff.

## Merge Criteria

A change can be merged when it:

- Improves public infrastructure, documentation, accessibility, testing, fake/sample data, or sandbox build readiness.
- Does not include real community data, member data, cultural material, real voting data, or sensitive personal information.
- Does not imply a real consultation, mandate, or community decision.
- Keeps Barayamal/Dean Foley's initial developer role clear without confusing it with community authority.

## Hold Criteria

A change should be paused when:

- It affects governance claims.
- It changes data handling rules.
- It introduces external dependencies or hosting assumptions.
- It may affect privacy, data sovereignty, cultural safety, or trust.
- It needs advisor review.

## Remove Criteria

Content should be removed immediately when it includes:

- Real names or records that are not explicitly fake/sample data.
- Cultural material.
- Member lists.
- Real eligibility or voting data.
- Sensitive personal information.
- Claims that the demo has made a real decision.
