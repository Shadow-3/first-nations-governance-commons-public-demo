# Fake-Data Boundary Audit v1.9

Date: 2026-06-04  
Mode: public demo and fake-data Decidim sandbox only

## Audit Rule

Only fake/sample data belongs in the public demo, repository, issue comments, screenshots, evidence logs, and sandbox.

## Excluded Content

Do not include:

- Real community records.
- Member lists.
- Personal information.
- Cultural material.
- Sacred, gendered, family, clan, or place-based restricted knowledge.
- Real eligibility data.
- Real voting data.
- Real decision outcomes.

## Review Routine

1. Review public pages.
2. Review Markdown documents.
3. Review CSV and JSON data.
4. Review screenshots before publishing or attaching to issues.
5. Review GitHub issue comments.
6. Review Decidim sample content.
7. Record any finding as a blocker.

## Pass Condition

The audit passes only when all public and sandbox-facing material is fake/sample data and the authority boundary is clear.

## Fail Condition

The audit fails if any real record, restricted knowledge, personal information, or binding decision claim appears.
