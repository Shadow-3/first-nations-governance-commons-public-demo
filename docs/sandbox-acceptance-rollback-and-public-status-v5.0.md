# Sandbox Acceptance, Rollback, and Public Status v5.0

Use this to decide whether execution work is safe to continue.

## Acceptance

Static demo checks may pass publicly, but private operational gates still need owner acceptance before any real participant or live Decidim action.

## Rollback

Rollback triggers include missing authority, privacy failure, IDS/data lifecycle failure, support gap, official-boundary breach, outbound real email, credentials/log leak, provider real-data request, and public launch wording.

## Public Status

Use approved hold wording only. Public copy must never imply applications are open, a live Decidim app exists, official Toastmasters workflows are active, or a private pilot is approved.

## Next Review

Reconvene owners when critical evidence arrives, provider P0 responses arrive, or blockers cannot be resolved.
