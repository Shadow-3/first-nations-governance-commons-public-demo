# Decidim Admin Configuration Checklist v1.6

Use this checklist after the sprint issues are opened and before the sandbox is shown as ready.

## Organisation

- Organisation name uses Barayamal / First Nations Governance Commons public demo wording.
- Public description says fake/sample data only.
- Locale, timezone, and contact details are set.
- Footer or help text links to the fake-data boundary.

## Assembly

- Assembly name clearly identifies sample governance context.
- Scope says draft/sample only.
- Members, if shown, are fictional roles or placeholders.
- No real community authority is implied.

## Participatory Process

- Process name matches 2026 Community Priorities and Budget sample.
- Phases match information, input, review, prioritisation, decision, and accountability.
- Process description includes offline participation and evidence capture.
- Process status does not imply a live decision.

## Components

- Proposals component has sample proposals only.
- Meetings component has sample yarning circle records only.
- Survey component has sample questions only.
- Page/accountability component has sample mandate and tracker content only.
- Moderation/reporting is enabled where available.

## Safety And Review

- Fake-data warning is visible from public pages.
- Admins can remove unsafe content quickly.
- No uploads contain personal records, cultural material, or real eligibility information.
- Known limitations are recorded before handover.

## Done

The sandbox is ready only when the Ops Room gates show Ready, the QA scenarios pass, and the evidence log is complete.
