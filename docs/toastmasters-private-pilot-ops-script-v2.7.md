# Toastmasters First Nations Private Pilot Ops Script v2.7

This script is a controlled operations rehearsal for the Toastmasters First Nations sample. It is not an official Toastmasters workflow and must not store real official records in the public demo.

## Script Goal

Prove that the pilot team can run intake, support, official handoff, incidents, evidence, and closeout without leaking private data or confusing the official Toastmasters charter process with Governance Commons public evidence.

## Rehearsal Steps

1. Start the run and record the named authority.
2. Confirm every test record uses a fake ID.
3. Route a demo-first participant to a minimal path.
4. Route a founding-member candidate to official Toastmasters handoff outside the public package.
5. Route a supporter to referral or mentoring without member-data access.
6. Route an access-support request to the named support role only.
7. Test a withdrawal request and record the closeout action.
8. Test a correction request and record the decision owner.
9. Trigger a fake incident where a private field appears in the wrong place.
10. Remove the unsafe evidence, notify the reviewer, and record closeout.
11. Produce a public evidence screenshot with fake IDs only.
12. Record the go, pause, or close decision.

## Expected Results

- Demo-first records are never counted as charter-ready.
- Founding-member candidates are handed to the official Toastmasters process outside the public package.
- Payment and sponsorship checks remain external.
- Access needs are visible only to the support role.
- Organisation supporters do not receive member data.
- Public evidence uses fake IDs and aggregate summaries only.
- Incidents have owner, response, containment, and closeout evidence.
- The pilot can close with export, retention, and deletion proof.

## No-Go Result

Stop immediately if the run produces real private data in any public file, screenshot, issue, CSV, JSON, or fake-data sandbox.
