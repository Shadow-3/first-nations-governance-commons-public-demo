# Decidim Permission And Authorisation Matrix v3.6

This matrix separates participation, administration, advisory review, developer custody, official-route ownership, and private authority.

## DBP-001 - Public visitor

Status: Allow

Allowed: View public-safe pages, proposals, meetings, updates, release notes, and fake sample data.

Limit: Cannot create, vote, see private evidence, or infer authority approval.

## DBP-002 - Registered participant

Status: Restricted

Allowed: Comment or propose only where the process allows it and public terms are accepted.

Limit: No real pilot participation until private approval.

## DBP-003 - Verified participant

Status: Restricted

Allowed: Can support, vote, or submit in restricted components after approved fake/private authorization.

Limit: No public ID document upload.

## DBP-004 - Local admin

Status: Restricted

Allowed: Manage assigned spaces, components, proposals, meetings, pages, and accountability items.

Limit: Cannot approve authority, privacy, IDS, or official records alone.

## DBP-005 - First Nations authority chair

Status: Restricted

Allowed: Chairs private motions, conditions, expiry, go/hold/no-go, and stop-rule decisions.

Limit: Private authority evidence never public.

## DBP-006 - Elder or cultural authority

Status: Restricted

Allowed: Advises on process legitimacy, cultural safety, and meeting protocol where invited.

Limit: Cultural material never public by default.

## DBP-007 - Facilitator

Status: Restricted

Allowed: Creates meeting summaries and offline import records using approved public-safe templates.

Limit: No private attendance or access-needs details in public notes.

## DBP-008 - Dean advisory reviewer

Status: Advisory

Allowed: Reviews public package, asks questions, raises findings, and checks build feasibility.

Limit: Cannot approve pilot or receive unrestricted private evidence.

## DBP-009 - Barayamal developer admin

Status: Restricted

Allowed: Builds, configures, tests, and documents fake-data environment as initial developer/custodian.

Limit: Developer custody is not First Nations authority.

## DBP-010 - Provider fake-data reviewer

Status: Restricted

Allowed: Reviews public build pack, fake seed data, and acceptance tests for quoting or delivery.

Limit: No production export or private data.

## DBP-011 - Toastmasters official-route owner

Status: Restricted

Allowed: Keeps official forms, member numbers, payments, signatures, and charter pathway outside demo.

Limit: Official records not stored in Decidim unless separately approved.

## DBP-012 - Data reviewer

Status: Restricted

Allowed: Approves data classes, access, retention, deletion, reporting, export, and return route.

Limit: Private evidence only.

## DBP-013 - Privacy reviewer

Status: Restricted

Allowed: Approves notice, consent, withdrawal, correction, minimisation, and incident response.

Limit: No real intake until privacy decision passes.

## DBP-014 - Safety reviewer

Status: Restricted

Allowed: Approves pause trigger, support route, incident handling, and affected-person support.

Limit: Incident details never public.

## DBP-015 - Participant manager

Status: Restricted

Allowed: Handles impersonation/support workflows if enabled for admin assistance.

Limit: Use only after privacy and access approval.

## DBP-016 - Unapproved external partner

Status: Blocked

Allowed: May read public package only.

Limit: No admin, evidence, exports, messages, or participant access.


Dean Foley / Barayamal remains advisory and initial developer/custodian only. That role can review the public blueprint and raise findings, but it cannot approve authority, privacy, IDS, official Toastmasters records, or unrestricted private evidence access.
