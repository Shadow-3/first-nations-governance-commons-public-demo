# Implementation Sprint Plan v1.3

Date: 2026-06-04  
Initial developer/custodian: Barayamal / Dean Foley  
Mode: fake-data Decidim sandbox sprint plan

## Purpose

This sprint plan gives a practical build sequence for the fake-data Decidim sandbox.

## Sprint 0 - Confirm Build Inputs

- Confirm sandbox is fake/sample data only.
- Confirm provider/developer access.
- Confirm hosting option.
- Confirm Decidim version and deployment method.
- Confirm acceptance test plan.
- Confirm no real data is included.

## Sprint 1 - Environment And Organisation

- Provision hosting or local build environment.
- Install and configure Decidim.
- Create organisation.
- Add public explanation pages.
- Add fake-data warning labels.
- Configure admin accounts and roles.

## Sprint 2 - Spaces And Components

- Create public assembly.
- Create participatory process.
- Configure proposals.
- Configure meetings.
- Configure surveys.
- Configure accountability/tracking approach.
- Configure moderation and public visibility settings.

## Sprint 3 - Sample Data And Review

- Import or manually create sample proposals.
- Import or manually create sample meetings.
- Add sample survey questions.
- Add sample accountability items.
- Check public labels and safety boundaries.
- Run advisor/developer walkthrough.

## Sprint 4 - Acceptance And Handover

- Run acceptance test matrix.
- Fix critical issues.
- Document known limitations.
- Complete admin handover.
- Confirm support and maintenance path.
- Decide whether to keep sandbox public, private, or hybrid.

## Exit Criteria

The sprint is complete when:

- Fake-data sandbox is running.
- Core demo flow is represented in Decidim.
- Acceptance tests pass or exceptions are documented.
- Handover notes are complete.
- No real community data has been introduced.
