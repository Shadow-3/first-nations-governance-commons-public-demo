# Sandbox Ops Runbook v1.6

Project: First Nations Governance Commons  
Mode: fake-data Decidim sandbox operations  
Initial developer/custodian: Barayamal / Dean Foley

## Operating Rule

Run the sandbox as infrastructure rehearsal only. Do not use real community records, member lists, cultural material, eligibility data, voting data, or real decision outcomes.

## Run States

- Ready: all fake-data, access, configuration, QA, evidence, and handover gates are passed.
- Hold: work can continue, but the sandbox should not be presented as ready.
- No-go: stop the sprint and resolve the blocker before continuing.

## Day 0 Setup

1. Confirm repository, sprint board, labels, and issue templates are active.
2. Confirm Decidim sandbox URL, admin access, support contact, and backup admin path.
3. Confirm fake-data banner and public safety labels are visible.
4. Confirm no real data or sensitive material is in the working set.

## Day 1 Configuration

1. Configure organisation settings, locale, timezone, and public labels.
2. Create the sample assembly and governance context.
3. Create the 2026 Community Priorities and Budget process.
4. Configure proposal, meeting, survey, page/accountability, and moderation components.
5. Add only fake/sample data from the approved import map.

## Day 2 QA

1. Run participant, admin, mobile, accessibility, and fake-data boundary scenarios.
2. Capture screenshot evidence for every acceptance scenario.
3. Log every failed scenario as a GitHub issue.
4. Stop immediately if real data, cultural material, or unsafe authority language appears.

## Day 3 Handover

1. Record admin handover notes, support contact, backup account status, and known limitations.
2. Export evidence log, QA scenario results, and open-risk register.
3. Decide Ready, Hold, or No-go for fake-data sandbox demo.
4. Record that no real pilot authority has been granted.

## Minimum Evidence

- Sandbox URL and admin dashboard screenshot.
- Fake-data warning screenshot.
- Assembly/process/component screenshots.
- Proposal, meeting, survey, and accountability scenario screenshots.
- Mobile viewport screenshot.
- Evidence log with owner and date.
- Known limitations and unresolved risks.

## Handover Statement

The sandbox may demonstrate Decidim configuration and fake/sample workflows. It must not be represented as a real consultation, a community mandate, or a production system.
