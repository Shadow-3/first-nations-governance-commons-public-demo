# GitHub Issue Pack v1.5

Use these issue titles and bodies to open the first public fake-data sandbox sprint issues.

## Issue Labels

Recommended labels:

- `sandbox`
- `fake-data`
- `decidim-config`
- `safety`
- `handover`
- `accessibility`
- `needs advisor review`

## Issue Template

Every issue should include:

- Task.
- Lane.
- Owner.
- Source file.
- Acceptance check.
- Fake-data boundary confirmation.

## Opening Order

1. Create sprint board.
2. Open setup issues.
3. Open Decidim configuration issues.
4. Open safety/QA issues.
5. Open handover issues.
6. Move blocked items into `Blocked by governance/safety`.

## Rule

An issue can organise infrastructure work.

It cannot approve real community authority.
