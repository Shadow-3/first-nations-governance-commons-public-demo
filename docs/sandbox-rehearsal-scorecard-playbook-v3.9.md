# Sandbox Rehearsal Scorecard Playbook v3.9

## Review finding

v3.8 created a controlled fake-data test lab, but the project still needed a decision-ready scorecard that turns rehearsal results into pass, hold, blocked, remediation, rollout, and public-summary decisions.

## Operating rule

Use the Sandbox Rehearsal Scorecard after the v3.8 Controlled Sandbox Test Lab. The scorecard converts fake-data rehearsal results into a decision-ready readout: what passed, what is blocked, what must be remediated, what can be publicly summarized, and what must remain private.

## How to use the scorecard

1. Run the v3.8 fake-data scripts.
2. Mark every result as Pass, Hold, or Blocked.
3. Convert every Hold or Blocked result into a remediation action.
4. Keep private evidence outside the public package.
5. Use the decision options to choose hold, second rehearsal, no-go, archive, provider handoff, or private sandbox build.
6. Publish only the public-safe summary items.
