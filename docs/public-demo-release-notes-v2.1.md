# Public Demo Release Notes v2.1

Release: v2.1 Fake Seed Studio  
Date: 2026-06-05  
Initial developer/custodian: Barayamal / Dean Foley

## Review Finding

v2.0 made the launch path clearer, but the package still needed a Decidim-shaped fake content seed layer. A builder could choose provider-assisted, self-build, partner-hosted, or hold-only, but still had to infer which spaces, components, records, and safety holds should be configured first.

- Added `seed-studio.html`.
- Added `docs/fake-data-seed-playbook-v2.1.md`.
- Added `docs/decidim-seed-admin-guide-v2.1.md`.
- Added `data/decidim-space-component-seed-v2.1.csv`.
- Added `data/fake-content-seed-register-v2.1.csv`.
- Added `data/seed-safety-review-v2.1.csv`.
- Added `data/fake-seed-package-v2.1.json`.

## What Is Stronger

- Four Decidim seed spaces are mapped: Governance Commons Assembly, Community Priorities And Budget 2026, Youth Digital Stewardship Sprint, and Services Feedback And Follow-up.
- Six official Decidim component types are represented across the seed package.
- Twenty-four fake content records are registered with source files, target components, status, and safety classification.
- Eight explicit safety holds block real records, cultural material, voting data, youth data, service cases, and authority claims.
- The interactive page generates a seed handoff brief for Barayamal, a provider, or a partner-hosted sandbox.

## Still Not Included

- No real Decidim instance is deployed by this release.
- No real import automation is claimed.
- No real community data, member records, cultural material, eligibility records, votes, supports, service cases, or outcomes are included.

## Recommended Next Step

Use the Sandbox Launcher first, then use Seed Studio to prepare the fake content handoff for the selected build path. Configure the Governance Commons Assembly and Community Priorities And Budget 2026 process before optional youth or services examples.
