# Dean Trusted Reviewer Brief v3.4

Dean Foley / Barayamal is used here as advisory and trusted reviewer for the public-safe package.

## Role

- Review public-safe materials.
- Identify gaps, no-go triggers, assumptions, and meeting questions.
- Help keep the public demo and private evidence boundary clear.
- Escalate unresolved blockers to the right owner.

## Boundary

- Dean's advisory review is not First Nations authority.
- Dean's developer/custodian role must be visible when giving advice.
- Dean should not receive unrestricted private evidence unless approved by the authority group.
- Dean should not approve real activation, provider handoff, or scale alone.

## Reviewer checklist

- RV-001: Confirm advisory boundary (Required) - State advisory role only; no final authority or sole approval
- RV-002: Conflict and custody note (Required) - Declare developer/custodian role and any review assumptions
- RV-003: Public data leak check (Required) - Check page, docs, CSV, JSON, screenshots, package files for real data
- RV-004: Authority evidence question list (Required) - List missing authority evidence for the private authority group
- RV-005: IDS/data governance question list (Required) - List unresolved data governance and community benefit questions
- RV-006: Privacy question list (Required) - List notice, consent, withdrawal, correction, retention, and incident questions
- RV-007: Official Toastmasters boundary check (Required) - Confirm official records stay outside public and private pilot tools unless approved
- RV-008: Provider fake-data boundary check (Hold) - Confirm provider scope uses fake/sample data only until contract/access approval
- RV-009: Support and incident escalation check (Hold) - Ask whether participant support and incident pause route is staffed
- RV-010: Reviewer notes redaction check (Required) - Remove real names, contact details, access needs, incidents, or private decisions from notes
- RV-011: No-go escalation note (Required) - Escalate any blocked trigger to authority, data, privacy, technical, safety, or official owner
- RV-012: Meeting readiness recommendation (Required) - Recommend public demo only, hold, or private authority meeting readiness
