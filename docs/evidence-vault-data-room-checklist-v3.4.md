# Evidence Vault Data Room Checklist v3.4

## Vault setup

- Approved private workspace.
- Named owner.
- MFA and least privilege.
- Access register.
- Backup/export/delete/restore proof.
- Evidence classification.
- Audit log.
- Revocation test.

## Evidence classes

- Public safe: status summaries and fake/sample package evidence.
- Private restricted: real governance, privacy, consent, support, provider, monitoring, and closeout evidence.
- Never public: access logs, incident facts, official records, signatures, payments, member numbers, authority records, and private decisions.

## Access rules

- AR-001: Dean advisory access (Private restricted) - Dean cannot receive unrestricted private vault access
- AR-002: Authority group access (Never public) - Names and access logs never public
- AR-003: Data reviewer access (Private restricted) - No official Toastmasters payment/member records
- AR-004: Privacy reviewer access (Private restricted) - No public screenshots of private records
- AR-005: Technical custodian access (Never public) - Technical proof stays private
- AR-006: Safety reviewer access (Never public) - Real incident facts never public
- AR-007: Provider access (Public safe) - No production export
- AR-008: Public reviewer access (Public safe) - No real names or decisions
- AR-009: Official Toastmasters access (External route) - Official records remain external
- AR-010: Support route access (Private restricted) - Access-needs details never public
- AR-011: Audit access (Never public) - Audit logs never public
- AR-012: Public status summary (Public safe) - Status only
