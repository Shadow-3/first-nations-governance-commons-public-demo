# Public Demo Release Notes v2.4

Release date: 2026-06-05

## Summary

v2.4 adds Build Delivery Control, a contract-to-sandbox workflow that sits after v2.3 Build Partner Desk.

The upgrade makes the package stronger by turning provider selection into controlled delivery: milestones, deliverables, evidence gates, payment release rules, RACI roles, no-go triggers, and a generated delivery brief.

## Why It Matters

The v2.3 provider selection workflow was useful, but it did not fully control what happens after a provider is chosen. v2.4 adds the missing delivery governance layer so Barayamal / Dean Foley can keep fake-data scope, source-code access, acceptance evidence, payment release, handover, and real-pilot boundaries visible.

## Public Demo Boundary

All provider, contract, delivery, payment, and milestone data is fake/sample data. This release does not approve real procurement, real spend, real community authority, or real community data use.
