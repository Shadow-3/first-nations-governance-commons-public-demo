# Private Pilot Operations Runbook v2.7

Release: v2.7 Pilot Operations Control  
Mode: public template with fake/sample data only  
Sample initiative: Toastmasters First Nations  
Initial developer/custodian: Barayamal / Dean Foley

## Review Finding

v2.6 decides whether a controlled private pilot can start. The next gap is operating discipline after approval.

Without an operations layer, a pilot team could approve the readiness gates but still lose control during intake, access support, incident response, official Toastmasters handoff, public evidence capture, or closeout.

## What v2.7 Adds

- Sixteen operations checks for run-day control.
- Twelve participant state rules for private workflow handling.
- Ten incident and closeout rehearsal rows.
- Eight official handoff rules separating Toastmasters records from Governance Commons public evidence.
- A browser-generated pilot operations brief.
- Document viewer coverage for all v2.7 docs, CSVs, and JSON.

## Operating Rule

The public demo remains fake/sample only. The private pilot can only operate in a separate approved environment with approved privacy notice, consent text, minimum fields, role access, incident route, official handoff boundary, and closeout rule.

## Daily Sequence

1. Confirm run authority and no-public-data reminder.
2. Confirm the private intake notice is active.
3. Confirm role/access list and admin removal path.
4. Route each participant through the participant state map.
5. Keep official Toastmasters records in the official external workflow.
6. Capture only fake or redacted evidence for the public package.
7. Check incident triggers before publishing any update.
8. Record the daily continue, pause, or close decision.
9. Close records under the retention and deletion rule.

## Stop The Pilot If

- Any private field appears in public evidence, GitHub, screenshots, CSV, JSON, or the fake-data Decidim sandbox.
- Official applications, signatures, member numbers, charter roster, or payment records are stored in Governance Commons public files.
- Organisation supporters receive member-level, access-need, payment, or admin access.
- A withdrawal, correction, or access-support request has no owner.
- Public wording implies the proposed club is chartered or that someone is an official member before Toastmasters acceptance.
- The run continues without a same-day authority decision.

## Required Evidence

Before and during operations, retain private evidence for:

- Approved run authority.
- Privacy notice and consent text.
- Minimum field list.
- Role/access register.
- Incident route and response clock.
- Official Toastmasters handoff boundary.
- Redaction evidence for public screenshots.
- Closeout, export, retention, and deletion proof.

## Public Evidence Rule

Public evidence can show fake IDs, aggregate counts, redacted test results, and approved public summaries only. It cannot show real names, contact details, access needs, payment details, member numbers, official forms, meeting links, signatures, private organisation contacts, or real authority records.

## Next Recommended Move

Use v2.7 after v2.6 readiness approval and before any live private pilot operation. If all required checks pass, rehearse the full private workflow with fake IDs first. Only then consider a separate, approved private environment for real records.
