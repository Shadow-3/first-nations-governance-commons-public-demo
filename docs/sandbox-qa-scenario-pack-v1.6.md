# Sandbox QA Scenario Pack v1.6

Project: First Nations Governance Commons  
Mode: fake-data Decidim sandbox QA

## Test Rule

Every QA scenario must use fake/sample data only. Any real community record, personal data, cultural material, or real decision outcome is an immediate no-go.

## Scenario Groups

### Public Visitor

- Opens the public sandbox and sees fake-data warning.
- Opens the assembly and understands it is sample context only.
- Opens the participatory process and sees sample phases.
- Opens a proposal without needing a real identity.

### Participant Simulation

- Supports a sample proposal where the sandbox allows it.
- Opens a sample meeting/yarning circle record.
- Completes or previews a sample survey.
- Reads a sample accountability or mandate record page.

### Admin

- Finds the configured organisation, assembly, process, and components.
- Confirms moderation/reporting settings.
- Removes or edits unsafe sample content if needed.
- Exports or records evidence for handover.

### Mobile And Accessibility

- Core public pages fit a 390 px viewport without horizontal page scroll.
- Interactive controls are reachable by keyboard.
- Text contrast and focus states are acceptable for a demo review.

## Pass Evidence

Each scenario should have status, owner, date, screenshot/file reference, and any follow-up issue ID.

## No-go Evidence

Stop the sprint if QA finds real data, sensitive material, unsafe authority language, broken admin access, or a public page that could be mistaken for a real consultation.
