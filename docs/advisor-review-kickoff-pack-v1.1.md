# Advisor Review Kickoff Pack v1.1

Date: 2026-06-04  
Initial developer/custodian: Barayamal / Dean Foley  
Mode: public demo review with fake/sample data only

## Purpose

This pack helps Barayamal run a focused advisor review before commissioning a fake-data Decidim sandbox.

## What Advisors Should Review

Ask advisors to review:

- Public demo prototype.
- Review Pack hub.
- Decision memo.
- Governance charter draft.
- Data sovereignty charter draft.
- Privacy impact pre-assessment.
- Go/no-go scorecard.
- Decidim sandbox statement of work.

## Review Questions

1. Would this process be trusted as a real community decision pathway if governed properly?
2. What must never be uploaded, opened, or made public?
3. How should offline/yarning circle input be represented without exposing sensitive material?
4. What authority must approve a real pilot?
5. What eligibility rules are needed before any real vote or prioritisation?
6. What would funders or partners need to see before supporting the sandbox?
7. What should be removed or changed before a real pilot is discussed?

## Review Session Agenda

- 5 minutes: explain fake-data-only boundary.
- 10 minutes: walk through the public demo.
- 10 minutes: open the Review Pack and document viewer.
- 10 minutes: discuss risks, authority, access, and evidence.
- 10 minutes: complete go/no-go scorecard.
- 5 minutes: agree next action.

## Minimum Review Threshold

Before proceeding to sandbox build:

- At least 3 advisor responses should be logged.
- No automatic no-go condition should remain unresolved.
- Any serious governance, data, privacy, or cultural safety concern should be converted into a decision log item.

## Follow-Up

After each session:

- Update `data/advisor-review-tracker.csv`.
- Add unresolved issues to `data/decision-log-template.csv`.
- Summarise feedback in `docs/advisor-synthesis-template-v0.7.md`.
- Update sandbox scope if feedback changes the build requirements.
