# Community Authority Boundary v1.4

## Plain Rule

The repository can organise software work.

It cannot organise or replace community authority.

## Repository Decisions

Repository maintainers can decide:

- Static demo code.
- Documentation structure.
- Fake/sample data.
- Decidim sandbox mapping.
- Issue labels and templates.
- Public release packaging.
- QA and deployment process.

## Community Authority Decisions

Community authority must decide:

- Whether a real pilot should exist.
- Who can participate.
- Who can vote or prioritise.
- What can be published.
- What data can be collected.
- What cultural material is excluded or restricted.
- Who can represent outcomes.
- Who can approve partner or funder use.

## Practical Guardrail

If a decision would affect real people, real records, real authority, real culture, or real services, it does not belong to repository maintainers alone.
