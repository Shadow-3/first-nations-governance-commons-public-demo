# Two-Track Pilot Onboarding Playbook v3.7

Use this playbook after the Decidim Build Blueprint.

## Review finding

v3.6 made the Decidim build configuration clear, but the next model needed a concrete onboarding plan: Barayamal as the First Nations organisation test and First Nations Toastmasters as a controlled member tester.

## Operating rule

Barayamal is the primary First Nations organisation onboarding test for Indigenous founders, business owners, mentors, and program participants. First Nations Toastmasters is a smaller controlled member onboarding tester. Keep the tracks separate, fake-data-only, and gated until private authority, privacy, IDS/data governance, support, official-route, and environment approvals pass.

## Advisory boundary

Dean Foley / Barayamal can act as initial developer, custodian, and advisory trusted reviewer for the fake-data onboarding model, but cannot approve First Nations authority, privacy, Indigenous Data Sovereignty, official Toastmasters records, participant eligibility, or private member/founder data access.

## Two-track sequence

1. Keep Barayamal and First Nations Toastmasters in separate Decidim spaces/components.
2. Seed fake records only.
3. Test Barayamal founder/business owner onboarding before any real founder records.
4. Test First Nations Toastmasters member/guest onboarding without official Toastmasters records.
5. Run redaction, mobile, admin revoke, official-boundary, and support-route tests.
6. Hold real invites until private authority, privacy, IDS/data governance, official-route, support, backup/restore, and closeout decisions pass.
