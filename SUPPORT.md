# Support

## Public Demo Support

Use GitHub issues for public demo bugs, documentation improvements, fake-data sandbox tasks, accessibility issues, and implementation planning.

Do not use GitHub issues for real community disputes, private records, cultural material, member data, or urgent safety matters.

## Current Support Role

Initial developer/custodian: Barayamal / Dean Foley.

## Response Priority

Highest priority:

- Unsafe data exposure.
- Misleading public claims.
- Broken public demo pages.
- Accessibility or mobile blockers.
- Fake-data boundary issues.

Normal priority:

- Documentation improvements.
- Sandbox implementation tasks.
- UI polish.
- Sample data corrections.
